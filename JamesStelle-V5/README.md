# 关于Carrier的逻辑判定（看你自己）
1. creep.store.getUsedCapacity() === 0<br>
含义：检查 Creep 的存储是否完全为空。<br>
适用场景：<br>
严格空载时才取能：确保 Creep 只在没有任何资源时才执行取能逻辑。避免中间状态干扰：适合需要明确区分“空”和“非空”状态的场景。<br>
优点：逻辑清晰，直接对应“空载”状态。避免 Creep 在存储未满时重复取能。<br>
缺点：如果 Creep 携带少量能量（未满），不会触发取能逻辑，可能导致效率降低。<br>
2. creep.store.getFreeCapacity() > 0<br>
含义：检查 Creep 的存储是否有空余容量（未满）。<br>
适用场景：未满时持续取能：适合需要 Creep 尽可能多携带资源的场景。<br>
动态调整：允许 Creep 在未满时继续取能，提高效率。<br>
优点：资源利用率更高，Creep 会尽量填满自己。适合需要快速搬运的场景。<br>
缺点：可能导致 Creep 在搬运过程中频繁切换目标（如果资源点分散）。<br>
